# 网络优化

# DNS 优化

当我们访问 https://www.taobao.com/ 淘宝的时候，首先需要把域名转化为对应的 IP 地址，DNS 本身的解析是一个非常耗时的过程，它首次会经过浏览器，计算机， host是 中查找是否有缓存，如果有直接拿这个 IP 地址发起请求，如果没有会通过网络进行 DNS 查询。那么如何减少解析时间呢，我们可以使用DNS Prefetch(DNS 预解析)。

DNS Prefetch 通过提前解析用到的一些常见域名，大大减少了实际访问时所花费的时间

那 DNS Prefetch 是如何工作的呢？其实，浏览器会在空闲时间提前将这些域名转化为对应的 IP 地址，这里为了防止 DNS Prefetch 阻塞页面渲染影响用户体验，Chrome 浏览器的引擎并没有使用它的网络堆栈去进行预解析，而是单独开了8个完全异步的 Worker 线程专门负责 DNS Prefetch，所以 DNS Prefetch 不会影响首屏加载。

# 如何使用

浏览器自动解析

浏览器引擎在解析 HTML 页面的时候，会自动获取当前页面所有的 a 标签 herf 属性当中的域名，然后进行 DNS Prefetch。这里需要注意的是如果页面是 HTTPS ，那么浏览器为了确保安全是不会开启自动解析的，这个时候就需要在页面头部添加如下的meta标签：

<meta http-equiv="x-dns-prefetch-control" content="on">
1

以上就是手动开启浏览器自动解析 DNS，如果不想开启，将 content 置为 off

手动解析

直接添加如下link标签即可

<link rel="dns-prefetch" href="//www.taobao.com">
1

Tips: DNS Prefetch 解析后的域名对应关系会缓存到本地，一般本地缓存的数量为50~200个

# 传输加载优化

# preload 和 prefetch

link 标签 preload, prefetch 属性调整资源加载优先级，可以优化网站性能

  • preload: 提前加载较晚出现，但对当前页面非常重要的资源
  • prefetch：提前加载后继路由需要的资源，优先级低，浏览器空闲时间加载

介绍一下 link 标签的属性和属性值

属性 属性值
href 链接地址 url
ref stylesheet、preload、prefetch
as 加载资源类型 script、style、image、font、crossorigin

# pre-rendering 预渲染

打包时提前渲染页面，没有服务端参与

# GZip

使用 GZip 压缩传输可以优化网络传输

# Keep-Alive

网络频繁的建立连接是很耗时的，需要经过 TCP, SSL 等过程，使用 keep-alive 持久化方式，可以减少连接创建次数

# 避免重定向

页面从定向会很影响页面的性能，所以尽量避免

# CDN

将一些静态资源通过 CDN 的方式访问，不仅可以缩短访问时间，而且传输会经过压缩等优化

# service-worker

service-worker 支持离线访问，利用缓存能加速重复的速度

使用注意：

  • 延长了首屏时间，单页面总加载时间减少
  • 兼容性
  • 只能在 localhost 或 https 下使用

# HTTP2

使用 HTTP2 可以优化我们的网络传输，它有以下优点

  • 请求响应多路复用。同一时刻发起多个请求
  • 二进制传输
  • 压缩请求头
  • 服务器推送

# SSR

使用 SSR 服务端渲染，前端只是做页面的展示，减少资源的下载和网络请求，达到了优化的效果
