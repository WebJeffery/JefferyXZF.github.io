# 语言介绍

JavaScript 官方名称是 ECMAScript 是一种属于网络的脚本语言,已经被广泛用于Web应用开发,常用来为网页添加各式各样的动态功能,为用户提供更流畅美观的浏览效果。

1995年2月 Netscape 的布兰登.艾奇开发了针对网景公司的 Netscape Navigator浏览器的脚本语言 LiveScript。之后 Netscape与 Sun公司联盟后 LiveScript 更名为JavaScript 。

微软在 javascript 发布后为了抢占市场推出了 JScript。为了让脚本语言规范不在混乱，根据javascript 1.1版本推出了 ECMA-262 的脚本语言标准。

ECMA是欧洲计算机制造商协会由 Sum、微软、NetScape公司的程序员组成。

# 适用场景

  • 浏览器网页端开发
  • 服务器后台语言 Node.js
  • 移动端手机APP开发，如Facebook的 React Native、uni-app
  • 跨平台的桌面应用程序，如 electronjs

# 避免延迟

如果将 js 放在 <header> 标签中，浏览器会首先下载 js 资源并执行，这样会阻塞页面的渲染，同时如果下载和执行的脚本非常耗时，会导致很长时间的页面白屏问题。

延迟加载的事例

<head>
    <meta charset="UTF-8">
    <script src="脚本文件1"></script>
    <script src="脚本文件2"></script>
    <script src="脚本文件3"></script>
</head>

<body>
    <h1>Hello world</h1>
</body>

# 严格模式

严格模式可以让我们及早发现错误，使代码更安全规范，推荐在代码中一直保持严格模式运行

主流框架都采用严格模式，严格模式也是未来JS标准，所以建议代码使用严格模式开发

# 基本差异

# 命名规则

JS中的变量是弱类型可以保存所有类型的数据，即变量没有类型而值有类型。变量名以字母、$、_ 开始，后跟字母、数字、_

下面是合法的命名

let name = '张三';
let $='李四'
1
2

JS 语言关键字不能用来做变量名，比如 true、if、while、class 等。

let class = 'class name';

# 变量声明

有多种方式定义变量比如 var、let、const

  • var 声明的变量会出现变量提升，在全局声明会挂载在 window 对象上
  • let、const 必须先声明后使用，声明的变量出现暂时性死区(TDZ)，形成块级作用域，不会存在变量提升的情况，并且不能重复声明变量，在全局声明的变量不会挂载在 window 对象上
  • const 声明的变量要初始化，否则报错。基本类型不允许再次全新赋值，引用类型可以修改属性值，但不能改变引用的地址
